This version can manage multiple slides and export a full presentation with the same Slides button + slide-map overlay pattern as your decision-trees deck. It also still exports one-slide snippets.

If the mode is panel or plain, this uses the same line prefixes as the left column. If the mode is custom, paste raw HTML. Inline math like $\nabla L$ and displayed math like \( \frac{dL}{dw} = x \) render in preview and export.

How to write content

Use LaTeX-style content blocks:

\paragraph{Teach the pattern before teaching every symbol.}

\begin{itemize}
\item First key point
\item Second key point
\end{itemize}

\begin{enumerate}
\item A
\item B
\end{enumerate}

\begin{equation}
\nabla_w L &= \frac{\partial L}{\partial z}\frac{\partial z}{\partial w}\\
&= \frac{\partial L}{\partial z}\frac{\partial z}{\partial w}
\end{equation}

\begin{card}{Key idea}
Each neuron turns an incoming sensitivity into an outgoing sensitivity.
\end{card}

Inline math still works as $x^2$.

Inside \begin{equation}...\end{equation}, multi-line alignment with & and \\ is supported automatically.

Older slides can still use the legacy shortcuts too.

For a pseudocode block, choose Pseudocode block as the content type and paste plain indented pseudocode text.
