MotivationFrom kNN to regions

Why not just keep all the neighbors?

The notes motivate trees by contrasting them with k-nearest neighbors: kNN can model nonlinear boundaries, but it stores the whole training set, slows down at test time, and depends on a good distance metric.

Key intuition

  • If a test point lands in a region where almost all nearby training examples share one label, we do not need to remember every neighbor individually.
  • Decision trees compress that idea into a sequence of threshold tests that divide space into increasingly pure regions.
  • After training, prediction only requires traversing the tree to a leaf and reading off the majority label.
Trees trade “remember every example” for “remember a compact partition of the space.”

Greedy inductionPurity + compactness

The ideal tree is compact and pure

The notes ask for a tree that is maximally compact and whose leaves are pure. Finding the minimum-size consistent tree is NP-hard, so practical learning uses a greedy impurity-reduction strategy instead.

Goal 1: compact

Use as few nodes as possible while still separating the labels well.

Smaller tree \(\Rightarrow\) lower storage, simpler test-time path, easier to inspect.

Goal 2: pure leaves

Each leaf should contain mostly one label, ideally exactly one.

Leaf impurity near \(0\) means the leaf prediction is reliable.

Reality: greedy search

Try many candidate feature-threshold splits and choose the one that most reduces impurity right now.

best split = arg min weighted child impurity

Greedy template

  1. At a node, enumerate candidate splits \((f,t)\).
  2. Send points left if \(x_f \le t\), right if \(x_f > t\).
  3. Compute the weighted impurity of the two children.
  4. Pick the split with the smallest weighted impurity.
  5. Recurse until leaves are pure or no useful split remains.

Consistency question

A consistent tree exists exactly when there are no duplicate feature vectors carrying conflicting labels. If identical inputs have different labels, no deterministic tree can classify them both correctly.

This is one reason real pipelines often regularize depth or prune rather than forcing a full-depth exact fit.

ImpurityClassification trees

How do we measure “purity”?

For a subset \(S\), let \(p_k\) be the fraction of examples in class \(k\). Two common choices are Gini impurity and entropy.

Gini impurity

\[ G(S)=\sum_{k=1}^{c} p_k(1-p_k) \]

Binary case intuition: maximum impurity occurs around a 50/50 class mix, and impurity drops to \(0\) at a pure leaf.

Entropy

\[ H(S)=-\sum_{k=1}^{c} p_k\log p_k \]

Entropy is another way to penalize uncertainty. Pure leaves have zero entropy; balanced leaves have larger entropy.

Split objective

\[ I_T(S)=\frac{|S_L|}{|S|}I(S_L)+\frac{|S_R|}{|S|}I(S_R) \]

Choose the split that minimizes the weighted average impurity of the left and right children.

Teaching takeaway

  • Impurity is local: it only sees the child leaves produced by the next split.
  • That makes training efficient, but also explains why trees can be myopic.
  • Gini and entropy often lead to similar trees in practice.

ID3Recursive splitting

ID3 in one slide

The notes present ID3 as a recursive algorithm with simple base cases and a greedy split rule.

Base cases

  • If all examples in \(S\) have the same label, return a leaf with that label.
  • If no distinct attribute values remain to separate the examples, return a leaf with the mode label (or mean for regression).
Notice what is not a base case in the notes: we do not stop just because the next split fails to improve impurity.

Recursive step

\[ S_L=\{(x,y)\in S:x_f\le t\}, \qquad S_R=\{(x,y)\in S:x_f>t\} \]
  1. Try candidate features \(f\) and thresholds \(t\).
  2. Pick the split minimizing weighted child impurity.
  3. Apply ID3 recursively to \(S_L\) and \(S_R\).

Why not stop when no immediate impurity improvement happens?

Because some useful structures only emerge after multiple coordinated splits. The XOR pattern is the canonical example: no single first split solves it, but a depth-2 tree does.

Failure modeXOR

XOR exposes the tree’s myopia

A depth-2 tree can represent XOR perfectly, but the first split does not necessarily reduce impurity. Greedy local optimization can therefore struggle.

Teaching interpretation

  • Greedy split selection only looks one step ahead.
  • XOR needs a combination of two splits before purity appears.
  • So “no immediate gain” does not imply “no useful future subtree.”
Trees are powerful, but their standard training rule is myopic. Ensembles often help overcome this weakness.

CARTRegression trees

CART: the same structure, but with continuous targets

For regression, leaves predict numeric values. The notes use squared loss as impurity and predict the mean label in each leaf.

Regression-tree impurity

\[ L(S)=\frac{1}{|S|}\sum_{(x,y)\in S}(y-\bar{y}_S)^2 \]

Here \(\bar{y}_S\) is the average label in subset \(S\). A split is good if it creates children whose values are tightly concentrated around their own means.

Prediction at a leaf

\[ \hat{y}(x)=\bar{y}_{\text{leaf}(x)} \]

The learned function becomes piecewise constant: each leaf corresponds to a region with its own average response.

Classification view

Leaf stores a class label or class histogram.

output = majority label

Regression view

Leaf stores a real number, usually the mean of the training targets in that leaf.

output = local mean

GeneralizationDepth matters

Depth helps fit the training set - and can overfit it

The notes point out that deeper ID3 trees tend to overfit. They also place decision trees on the parametric/non-parametric spectrum depending on whether depth is fixed.

Overfitting intuition

  • As depth increases, leaves become smaller and can memorize idiosyncratic noise.
  • Training error usually falls monotonically.
  • Test error may first fall and then rise.
This motivates validation, pruning, minimum-leaf-size constraints, and ensembles.

RecapTakeaways

What students should leave with

Conceptual

  • A decision tree is a recursive partition of feature space.
  • Training is greedy local impurity reduction.
  • XOR shows why greediness can be shortsighted.
  • CART extends the same template to regression with squared loss.

Practical

  • Deeper trees fit more patterns but can overfit.
  • Minimum leaf size and max depth are core regularizers.
  • Single trees are often weaker than ensembles, but extremely interpretable.
  • The demos make the partitioning story concrete.
